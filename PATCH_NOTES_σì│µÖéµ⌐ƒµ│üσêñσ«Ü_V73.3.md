# PATCH NOTES｜即時機況判定 V73.3

## 調整目的

外勤回報流程為：

1. 架設時填寫新場次架設日。
2. 收設備時補填該場次後校正日。
3. 若移動至下一點，會新增下一筆新場次架設日。
4. 若後校正完成後收回保養，則不會新增下一場。

因此即時機況不應以固定天數緩衝判斷，而應以設備生命週期判斷。

## 新判定標準

| 條件 | 判定 | 說明 |
|---|---|---|
| 最新架設場次未填後校正日 | 運作中 | 設備仍在該點監測 |
| 最新架設場次已填後校正日，且沒有下一筆新場次 | 維護中 | 設備已收回，可能保養或待派場 |
| 後校正日後已有新增架設場次，且最新場次未填後校正日 | 運作中 | 已移動到下一點並重新上線 |
| 無架設回報資料 | 維護中 | 無法確認上線，保守列維護 |

## 程式修改

- `server.js`
  - 新增外勤回報列內 `後校正日` 讀取。
  - 以每台設備最新場次判斷是否仍在線。
  - 取消 `LIVE_EQUIPMENT_WAIT_NEW_SESSION_DAYS` 兩日緩衝邏輯。
  - 後台與 LINE 顯示改為設備生命週期說明。

## 建議 Google Sheet 欄位

外勤回報 Sheet 建議至少包含：

- 機台編號
- 執行場次
- 日期／架設日期
- 執勤地點
- 行政區
- 後校正值
- 後校正日
- 送出時間／回報時間

## 使用方式

部署後於後台點選「同步外勤機況」，或在 LINE 輸入：

```text
即時機況
```

即可查看各機台是否上線。
